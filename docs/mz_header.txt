Notes
-----
- The "MZ" magic bytes are the initials of Mark Zbikowski, the Microsoft
  engineer who designed the format (circa 1981).

- e_cblp = 0 is a special case. Normally it holds the number of valid
  bytes in the last 512-byte page. Zero means the last page is entirely
  used (treat as 512).

- 976 relocations is typical of a compiler-generated real-mode program
  (Turbo C, Borland C++, Watcom, etc.) where every far pointer to a
  data or code segment must be patched at load time.

- e_maxalloc = 0xFFFF is the standard DOS idiom for "allocate all
  available memory", preventing other programs from loading above this
  one. Common in games that want exclusive memory access.

- e_sp = 0x0080 (128 bytes) is a very small initial stack. The program
  likely switches to its own larger stack early in startup.

- e_cs = 0x0000 and e_ip = 0x0000 means execution starts at the very
  first byte of the load image, which at file offset 0x1200 is the
  program's startup code.
